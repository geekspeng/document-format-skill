# 中文文档格式化 Skill

本 Skill 基于《[中文技术文档写作规范](https://www.fengchao.pro/blog/document-style-guide)》对中文 Markdown 文档进行格式化处理。

## 核心格式化规则

### 1. 标点符号（全角/半角）

| 场景 | 正确 | 错误 |
|------|------|------|
| 中文句子 | 全角句号 `。` | 半角句号 `.` |
| 中文逗号 | 全角逗号`，` | 半角逗号 `,` |
| 中文顿号 | 顿号`、` | 逗号 `,` |
| 中文引号 | 外层用「」，内层用『』 | "..." 或 '...' |
| 中文括号 | `（` `）` | `(``)` |
| 省略号 | `⋯⋯` | `...` 或 `......` |
| 破折号 | `————` | `---` |
| 叹号/问号 | `！` `？`，避免使用 | `!` `?`，不可 `！！` 连用 |

**重要规则：**
- 句号、逗号等点号不可出现在行首
- 引号嵌套时，外层用「""」，内层用『''』

### 2. 中英文混排空格

- **必须加空格**：全角中文与半角英文之间
  - 例如：`Vue 是优秀的框架` ✅
  - 例如：`Windows 用户` ✅
  - 例如：`16 GB`（数字与单位之间留适当空隙）
- **不加空格**：全角中文与半角数字之间不加空格
  - 例如：`性能提升50%` ✅
  - 例如：`用户量为1000万` ✅
- **不加空格**：中文标点与英文/数字之间
  - 例如：`Hello，world` ✅

### 3. 标题层级

- 一级标题（`#`）：文章主标题
- 二级标题（`##`）：主要章节
- 三级标题（`###`）：小节
- 四级标题（`####`）：尽量避免

**规则：**
- 层级必须连贯：一级→二级→三级，不可跳级
- 同级标题不应少于两个：避免一个二级标题下直接跟三级标题
- 下级标题不应与上级标题同名

### 4. 段落规范

- 一个段落只能有一个主题
- 段落长度不超过 7 行，最佳 ≤ 4 行
- 段落之间用空行隔开
- 段落开头不要留空白字符（不要首行缩进）
- 段首放中心句，对全段内容进行概述

### 5. 句子规范

- 单句尽量控制在 20 字以内，最长不超过 40 字
- 优先使用简单句和并列句，避免复合句
- 优先使用肯定句而非否定句
- 避免双重否定
- 正确区分`的`、`地`、`得`
- 使用主动语态，不用被动语态
- 避免非正式用语

### 6. 数字规范

- 阿拉伯数字一律使用半角：`1` `2` `3`
- 千分位用半角逗号：`1,000` `100,000`
- 数值范围用波浪线或一字线：`10～20` 或 `10—20`
- 增加用"增加了"（差量）或"增加到"（总量）
- 减少用"降低了"（差量）或"降低到"（总量）
- 货币：数字前加货币符号，或数字后写中文名称，如 `$1,000` 或 `1,000 美元`

### 7. 列表规范

- 列表项开头使用顿号、`、`或数字编号（1. 2. 3.）
- 列表内句子不需要句号结尾（简短条目）
- 避免列表嵌套过深（建议不超过 3 层）

### 8. 代码规范

- 行内代码用反引号 `` `code` ``
- 代码块指定语言便于高亮
- 命令行示例避免使用 `$` 或 `>` 前缀（用户会复制）

### 9. 英文处理

- 英文复数翻译成中文时使用单数形式：如 `APIs` → `API`
- 缩写可用半角圆点表示：如 `U.S.A.`
- 英文书名改用中文时用《》：如《The Art of Programming》
- 首次出现英文词汇时括号标注中文：如 `Redis（分布式缓存）`
- 专有名词首字母大写，非专有名词不大写

### 10. 文件名规范

- 文件名不得含有空格
- 必须使用半角字符（中文不能用于文件名）
- 建议只用小写字母
- 多个单词用半角连词线分隔：如 `advanced-usage.md`
- 特殊说明文件可用大写：如 `README.md`、`LICENSE.txt`

## 格式化流程

当用户请求格式化中文文档时，执行以下步骤：

### Step 1: 读取并分析内容

仔细阅读用户提供的文档内容，识别所有格式问题。

### Step 2: 逐项检查

按上述 10 类规则逐一检查：

1. **标点符号**：全角/半角检查
2. **空格**：中英文混排空格检查
3. **标题**：层级连贯性检查
4. **段落**：长度、空行、首行缩进检查
5. **句子**：长度、肯定/否定语气、被动语态检查
6. **数字**：半角、格式、货币检查
7. **列表**：格式一致性检查
8. **代码**：反引号、语言标记检查
9. **英文**：复数还原、缩写、首次标注中文
10. **文件名**：空格、半角字符、连词线分隔

### Step 3: 生成格式化结果

输出格式如下：

```markdown
## 格式化结果

[格式化后的完整文档]

---

## 修改说明

| 位置 | 问题 | 修改 |
|------|------|------|
| 第3行 | 中文与英文之间缺少空格 | 添加空格 → `Vue 是` |
| 第5行 | 逗号应为顿号 | `,` → `、` |
| ... | ... | ... |
```

### Step 4: 解释改动原因

向用户简要说明主要改动类别，帮助用户学习规范。

## 常见问题处理

### 问题：不确定是否需要空格

```
✅ 正确：使用 Vue.js 构建应用
❌ 错误：使用Vue.js构建应用
❌ 错误：使用 Vue . js 构建应用
```

### 问题：引号嵌套

```
✅ 正确：他说：「这个问题很简单。」
✅ 正确：外层用「」，内层用『』：「他说：『这是内层引号。』」
❌ 错误：他说："这个问题很简单。"
```

### 问题：省略号与其他符号连用

```
✅ 正确：功能包括⋯⋯
❌ 错误：功能包括⋯⋯等
```

### 问题：数字增减表述

```
✅ 正确：性能提升了 50%
❌ 错误：性能提升了 50 倍
```
